amux

Agent Multiplexer

Run dozens of Claude Code agents unattended. amux watches every session on a 60s loop and handles what would otherwise interrupt you: context compaction, thinking-block recovery, tool-approval prompts. Agents coordinate via a shared REST API — no explicit programming required.

Install & quickstart
$ git clone https://github.com/mixpeek/amux && cd amux && ./install.sh

# Register a project and start an agent
$ amux register api --dir ~/Dev/api --yolo
$ amux start api

# Open the web dashboard (or install as a PWA on your phone)
$ amux serve
  → https://localhost:8822

How it works

  • Self-healing watchdog — snapshots every session every 60s. Context low → /compact. Thinking-block corruption → restart + replay. Stuck waiting → auto-respond. All silent.
  • Truly headless YOLO mode — --dangerously-skip-permissions still shows blocking tool-approval prompts. amux detects them via the "Esc to cancel" marker (never appears on open-ended questions) and auto-answers, keeping sessions actually unattended.
  • Agent coordination out of the box — $AMUX_SESSION and $AMUX_URL are injected at startup. A global memory file (shared via REST) ships the full API reference — agents discover peers, peek output, and claim tasks without being told how.
  • Atomic task claiming — no broker — POST /api/board/:id/claim uses SQLite UPDATE … RETURNING as a CAS. Dozens of agents can race for the same queue with no Redis, no duplicated work.
  • Web dashboard + PWA — live session cards, peek mode, multi-pane workspace, kanban board with iCal sync. Installable on iOS/Android. Tailscale gives you real TLS and phone access with no config.
  • One Python file, zero build step — ~12k lines: Python ThreadingHTTPServer + full dashboard as inline HTML/CSS/JS. Edit it; it restarts itself via os.execv. Only deps: python3 and tmux.
