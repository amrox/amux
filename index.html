Why amux
Problems it solves
Every friction point when running multiple Claude Code agents at scale.
Claude crashes at 3am from context compaction
Self-healing watchdog — auto-compacts, restarts, replays last message
No way to monitor 10+ sessions from one place
Web dashboard with live status, token spend, and session peek
Agents duplicate work on the same tasks
Kanban board with atomic task claiming via SQLite CAS
No way to manage agents from your phone
Mobile PWA with offline support and Background Sync
Agents cannot coordinate with each other
REST API orchestration — send, peek, claim tasks between sessions
No persistent memory across restarts
Per-session and global memory files, auto-snapshotted logs
Capabilities
Everything you need
A complete control plane for AI coding agents.
sessions
Parallel Isolation
Run any number of Claude Code instances, each isolated in its own tmux pane with its own conversation history and working directory.
monitoring
Live Status Detection
Every session card shows working, needs input, or idle — updated in real time via Server-Sent Events. No polling required.
access
Web Dashboard + PWA
HTTPS dashboard installable as a home screen app. Monitor agents, send commands, and browse output from any device.
tasks
Kanban Board
SQLite-backed with auto-generated keys, atomic claiming, tags, due dates, drag-and-drop columns, and iCal export.
automation
REST API
Every action is an endpoint. Automate orchestration with curl or any HTTP client. Register, start, stop, send, peek.
security
Tailscale + TLS
Auto-provisions certs via Tailscale, mkcert, or self-signed fallback. Secure remote access without open ports.
input
File Attachments
Paste images with Ctrl-V, drag and drop files, or use the attach button. Supports images, PDFs, CSV, JSON, and logs.
persistence
Session Memory
Each agent gets its own persistent memory file, plus a shared global memory. Logs are snapshotted every 60 seconds.
cost
Token Tracking
Per-session daily spend with cache reads broken out separately. Reset counters anytime from the dashboard.
Dashboard
Your fleet at a glance
Session cards, peek mode, kanban board, and search — all in a single-page PWA.

CLI
One command, any action
Session names support prefix matching — amux a my resolves to myproject.
CommandAliasDescription
amux register <n> --dir <path>regRegister a new session
amux start <n>Start a session headless
amux stop <n>killStop a running session
amux attach <n>aAttach to a session's tmux pane
amux peek <n>pView output without attaching
amux send <n> <text>Send text or a command to a session
amux exec <n> -- <prompt>runRegister + start + send in one shot
amux lslistList all sessions
amux servewebStart the web dashboard
amux start-allStart all registered sessions
amux defaultsconfigManage default flags

orchestration example
# Send a task to another session
$ curl -sk -X POST \
-H 'Content-Type: application/json' \
-d '{"text":"implement login"}' \
$AMUX_URL/api/sessions/worker-1/send

# Atomically claim a board item
$ curl -sk -X POST \
$AMUX_URL/api/board/PROJ-5/claim

# Watch another session's output
$ curl -sk "$AMUX_URL/api/sessions/worker-1/peek?lines=50"
